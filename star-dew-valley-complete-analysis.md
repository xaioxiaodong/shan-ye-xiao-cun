## 1.2 世界观设定

```
你厌倦了城市里无聊的办公室生活。
在你祖父去世后，留下了一封信：
"当你觉得被现代生活的重担压垮时，
打开这封信……星露谷在等着你。"
```

- **时代：** 现代（有电视、电脑、冰箱等现代电器），但小镇保留田园风貌
- **地点：** 虚构的 Stardew Valley（星露谷），位于某乡村地区
- **氛围：** 温暖、舒适、带有些许奇幻元素（魔法、矮人、幽灵）
- **Joja 公司：** 游戏中的"反派"——大型企业侵蚀小镇文化的象征

## 1.3 核心体验目标（Emotional Goals）

| 目标 | 实现方式 |
|------|----------|
| **平静感** | 没有强制任务，没有时间压力（除了体力限制） |
| **成就感** | 从荒芜农场到自动化生产，每一步都是肉眼可见的进步 |
| **归属感** | NPC 从陌生到熟悉，小镇从破败到繁荣 |
| **探索欲** | 矿洞无止境、秘密无处不在 |
| **掌控感** | 从体力/金钱都紧缺到资源自由 |

## 2.2 各区域详解

### 2.2.1 你的农场（Farm）

**进入条件：** 游戏开始即进入

**农场布局：** 共 7 种农场地图（版本 1.6 新增 1 种）

| 农场类型 | 特点 | 难度 |
|----------|------|------|
| **标准农场** | 最大可耕种面积，最多空地 | ⭐ |
| **河畔农场** | 大片水域，适合钓鱼 | ⭐⭐ |
| **森林农场** | 树桩会再生，采集物多 | ⭐⭐ |
| **山顶农场** | 矿区+刷新矿石 | ⭐⭐⭐ |
| **荒野农场** | 夜间刷怪 | ⭐⭐⭐⭐ |
| **四角农场** | 适合多人，分区明确 | ⭐⭐ |
| **海滩农场** | 不能放洒水器（沙地），但海边拾取物多 | ⭐⭐⭐⭐⭐ |
| **草原农场 (1.6)** | 开局有鸡舍+两只鸡，草更茂盛 | ⭐⭐ |

**农场元素：**
- **可耕种土壤：** 数量取决于农场类型
- **不可耕区域：** 草地、池塘、悬崖、大树桩、巨石
- **初始清理对象：** 树根（橡树/松树/枫树）、圆木/大圆木、碎石/大石、杂草
- **农场山洞：** 可以选择"蘑菇洞"（每天产蘑菇）或"蝙蝠洞"（产水果）

### 2.2.2 鹈鹕镇（Pelican Town）

**子区域：**
| 子区域 | 包含 | 可进入建筑 |

| 小径1号/2号 | 部分 NPC 住宅 | — |

**关键建筑：**
- **皮埃尔的杂货店（Pierre's General Store）：** 买菜籽、基础工具、壁纸。09:00-17:00，周三休
- **诊所（Harvey's Clinic）：** 看病（25% 医保）、买药。09:00-15:00，周二休（急诊例外）
- **博物馆（Museum）：** 捐赠矿物/文物，每捐满一定数量解锁奖励
- **铁匠铺（Clint's Blacksmith）：** 工具升级、矿物加工、买煤/矿石。09:00-16:00

### 2.2.3 山脚区域（Mountains）

**关键位置：**
- **矿井入口（The Mines）：** 1-120 层，主线矿洞——详细见第五部分
- **木匠店（Carpenter's）：** 罗宾经营。造建筑/家具/购买资源。09:00-17:00（周二/雨休）
- **冒险家公会（Adventurer's Guild）：** 买武器/靴子/戒指，领取杀怪奖励。14:00-22:00（开馆仅当玩家进过矿井）
- **帐篷：** 莱纳斯（Linus）的家
- **铁路（Railroad）：** 温泉（恢复体力/能量）、铁轨（通往沙漠，需修好巴士）

### 2.2.4 煤矿森林（Cindersap Forest）

**关键位置：**
- **玛妮的牧场（Marnie's Ranch）：** 买动物（鸡鸭牛等），09:00-16:00（周一/二休）
- **法师塔（Wizard's Tower）：** 魔法类内容（建筑外观修改、结婚戒指等）
- **秘密森林（Secret Woods）：** 需要钢斧砍大圆木进入，内有硬木桩（可再生）、史莱姆怪
- **下水道入口：** 获得生锈钥匙后进入（捐赠满 60 件博物馆物品）。Krobus 商店
- **旅行商人（Traveling Cart）：** 周五/日出现在森林（卖稀有物品，价格随机）
- **废弃建筑物（Abandoned Building）：** 解锁 Joja 路线后变成电影院

### 2.2.5 沙滩（Beach）

**关键位置：**
- **Willy 的鱼店：** 买钓鱼竿、鱼饵、蟹笼。09:00-17:00（雨休）
- **码头：** 钓鱼好位置
- **船的残骸/潮池：** 低潮时可走到潮池区采集
- **老船（Old Boat）：** 修好后通往姜岛（版本 1.5+）

### 2.2.6 沙漠（Cali Desert）

**进入条件：** 完成巴士车站修理（社区中心 42,500g 捐献或 Joja 会员 40,000g）

- **三根柱子（Three Pillars）：** 放置彩虹贝壳触发特殊事件（获得银河剑）

# 第三部分：时间系统

## 3.1 时间流速

| 维度 | 数据 |
|------|------|
| 1 游戏日 | 约 13-14 分钟现实时间（不暂停） |
| 1 天游戏内时间 | 6:00 - 凌晨 2:00 |
| 玩家倒地时间 | 凌晨 2:00 之后强制昏迷 |
| 1 季节长度 | 28 天 |
| 1 年长度 | 4 季 × 28 天 = 112 天 |
| 存档格式 | 每天结束睡觉时自动保存 |

## 3.2 时间关键机制

| 机制 | 描述 |
|------|------|
| 6:00 AM | 玩家自动起床（疲劳恢复取决于睡眠时间） |
| 6:00 AM - 2:00 PM | 正常活动时间 |
| 凌晨 12:00 AM | 玩家开始变困，屏幕闪烁 |
| 凌晨 2:00 AM | 玩家强制昏迷倒地 |
| **睡眠奖励** | 12:00AM 前睡 → 精力全满。12:00-1:00AM → 部分精力。1:00-2:00AM → 精力减少 |
| **昏迷惩罚** | 丢失金钱（10% 丢失金额 + 上限 5000g + 不可丢鱼/工具/武器/戒指） |

## 3.3 季节系统

### 季节流转
```
春季 (Spring 1-28) → 夏季 (Summer 1-28) → 秋季 (Fall 1-28) → 冬季 (Winter 1-28) → 下一年春季
```

### 季节变化
| 方面 | 变化 |
|------|------|
| 地图外观 | 每个区域的颜色、植被、背景音变化 |
| 可种植作物 | 每个季节独有的作物（跨季作物会枯死） |
| 采集物 | 完全不同的采集品 |
| 鱼种 | 不同的鱼（某些鱼仅限特定季节+天气+时间） |
| 节日 | 每个季节 2-3 个独特节日 |
| NPC 日程 | 部分 NPC 季节性调整日程 |
| 天气 | 冬季不会下雨（只会下雪） |

## 3.4 天气系统

| 天气类型 | 效果 |
|----------|------|
| **晴天** | 默认，需要浇水 |
| **雨天** | 自动浇地，鱼更容易上钩，部分 NPC 呆在家 |
| **大风天** | 树/作物可能受损？(1.6 增加) |
| **风雪** | 冬季替代雨，效果同雨天 |
| **雷暴** | 雨天+闪电（劈作物/劈中树变成树液采集器电池棒） |
| **雪** | 仅冬季 |
| **节日** | 总是晴天 |

天气是**每天随机生成**的，但受以下影响：
- 电视天气预报可以预测下一天的天气
- "雨图腾"可以强制第二天降雨
- 社区中心完成部分后可影响天气

# 第四部分：农场经营系统（核心循环）

## 4.1 耕种基础（Tilling → Planting → Watering → Harvest）

### 耕地状态阶段

```
草地 → 用锄头耕地 → 种植+浇水 → 生长 → 成熟可收获
                                   ↓
                           种子 → (多次收获作物) → 新果继续
                                   ↓
                             收获 → 枯萎（茎清零）→ 重新耕地
```

### 耕地质量（1.6 新增机制）

使用锄头耕地时，耕地周围有概率出现**高质量地块**：
- 耕地后有概率出现"优质土壤"标记（星星闪烁）
- 在上面种植作物有几率直接产出高质量产物高概率

### 浇水
- 每次种植后必须浇水，作物才能生长
- **洒水器等级：** 见 4.6 自动化的故事
- 雨天自动浇水

## 4.3 作物质量系统

作物产出有 4 个质量等级，直接影响售价和 NPC 送礼效果（喜爱程度）。

| 等级 | 倍率 | 触发条件 |
|------|------|----------|
| 普通（Regular） | ×1.0 | 默认 |
| 银星（Silver） | ×1.25 | 有概率，受耕种等级+肥料影响 |
| 金星（Gold） | ×1.50 | 有概率，受耕种等级+肥料影响 |
| 铱星（Iridium，1.6新增） | ×2.0 | 有概率，受耕种等级+高品质肥料影响 |

**质量影响因素：**
- 耕种等级（每级 +3% 高质量概率，10 级时约 50%+ 金星）
- **肥料类型**（见 4.8）
- **耕种专精技能**（选择农耕家 10 级技能时所有作物自动铱星）

## 4.4 肥料系统

| 肥料 | 配方来源 | 材料 | 效果 |
|------|----------|------|------|
| **基础肥料** | 耕种 1 级 | 树液 ×2 | 提高土壤质量概率 ×（中等） |

# 第六部分：钓鱼系统

## 6.1 钓鱼机制

### 基础流程
```
1. 在岸边/船上装备鱼竿 → 蓄力抛线
2. ⏳ 等待鱼上钩（浮标下沉+音效）
3. 🎯 小游戏：浮动条控制在鱼图标范围内
   - 按住/松开鼠标 控制绿色条上下
   - 鱼跳动节奏变化（不同鱼不同模式）
4. 保持绿色条接触鱼直到进度条填满 → 钓鱼成功
5. 鱼脱钩（进度条变红）→ 钓鱼失败
```

## 11.2 节日详情

### 春 13——复活节
- 农场主人给镇民们举办找彩蛋活动
- **迷你游戏：** 在镇广场找到 9 个隐藏彩蛋（限时 50秒）
- **排名奖励：** 找到所有 9 个 + 冠军草帽
- **关键商店：** Pierre 卖 **草莓种子**（100g，只有这天买得到）
- 草莓是春季利润最高的作物（每 4 天收一次）

### 春 24——花舞节
- 与村民跳舞的浪漫节日
- 需要和舞伴好感度 4 心以上才能成功跳舞
- 不跳舞也可观赏，氛围很好
- 商店卖特殊花卉装饰

### 夏 11——夏威夷宴会
- **核心活动：** "锅汤"——每个村民往锅里扔食材
- 玩家也可以扔（+好感看好坏）
- **判断标准：** 投入的食材质量决定 Lewis 镇长品尝后的反应
  - ❌ 有毒/垃圾食物 → 负面反应（-好感）
  - ✅ 高品质食材 → 正面反应（全NPC +好感）
- 策略：带一件金星品质的食材

### 夏 28——月光水母节
- **纯情感节日：** 没有小游戏，没有竞赛
- 所有人聚集在沙滩，夜幕降临后看红色水母集体洄游
- **氛围设计：** 音乐变化+水母发光效果+温馨对话
- **用途：** 给玩家"停下来感受世界"的静心时刻

### 秋 16——星露谷展览会
- ⭐ **全游戏最重要的节日**
- **核心活动：** 展览展示（展示你的 9 件最高质量作物和动物产品）
  - 展示品品质越高 → 得分越高 → 星星币越多
- **小游戏：** 钓鱼、磨坊（强手棋）、打地鼠
- **商店：** 用星星币换种子、家具、稀有物品（星露谷头盔！）
- **各镇民给展示品打分机制：** 基于品质、多样性、季节作物适配

### 秋 27——万灵节
- **核心活动：** 迷宫探险（在森林里通过迷宫到达终点拿南瓜灯）
- 迷宫内有简单的解谜和小惊吓
- 商店卖金色南瓜、南瓜灯笼装饰

### 冬 8——冰雪节
- **核心活动：** 冰钓大赛
- 比赛时在特定位置钓到指定数量的鱼
- 获胜奖励：独特的捕鱼竿/装饰
- 其他活动：滑冰（免费）等

### 冬 15-17——夜市（版本1.5新增）
- **设定：** 码头有 3 晚的露天夜市
- **独特购物：**
  - 稀有种子（如 Rare Seed 甜宝石）
  - 魔法鱼（白天不出现的午夜鱿鱼）
  - 潜艇游览（看深海鱼 + 深水恐惧）
  - 画家（可定制各种装饰壁画）
  - 美人鱼表演（隐藏彩蛋）
- 这个节日**不影响时间流逝**（你在晚上可以慢慢逛，时间继续但是fish thing

### 冬 25——冬日星宴会
- **类似 Secret Santa：** 你抽到一位村民，要送他/她礼物
- 礼物价值影响对话反应
- 你也可以领到其他人给你的礼物

### 沙漠节（Desert Festival，版本1.6新）

**时间：** 每年春 15-17  
**地点：** 卡利科沙漠

# 第十八部分：游戏设计模式分析

## 18.1 星露谷为什么能让人"上瘾"？

### 心理学模型：Skinner Box + Self-Determination Theory

| 动机维度 | 游戏实现 |
|----------|----------|
| **自主感**（Autonomy） | 你想做什么，不做什么——种田、挖矿、谈恋爱都行 |
| **胜任感**（Competence） | 每次升级都是明确的正反馈 "你变强了" |
| **归属感**（Relatedness） | NPC 从冷淡到热情，从陌生到好友/恋人 |

### 时间管理 = 决策的快乐

```
每天 06:00 AM → 凌晨 2:00 AM 共约 18 小时
体力值 大约 270（初始）→ 依食物可扩展
```

- **存在约束但不强制**：你有体力和时间的双重限制，但"去睡觉"永远是随时可选的
- **先做什么后做什么**——本身就是有趣的策略决策
- **每一天都是新鲜的"有限资源分配问题"**

### 赌博心理机制（偶尔的惊喜）

| 惊喜类型 | 触发 | 心理效应 |
|----------|------|----------|
| 随机文物掉落 | 挖地/开晶洞/打怪 | 多巴胺↑ |
| 巨型作物 | 3×3 同种作物概率 | 哇！ |
| 矿井宝箱层 | 随机楼层 | "再下一层看看" |
| 秘密笔记 | 各种随机活动 | 这个世界还有秘密 |
| 旅行商人库存 | 周五/日刷新 | "今天有好东西吗？" |

## 18.2 内容解锁曲线

星露谷成功地将内容"分阶段释放"给玩家：

| 时间 | 解锁内容 | 驱动 |
|------|----------|------|
| 第1天 | 基础种田（防风草）+ 基础钓鱼/采集 | 活下去 |
| 第5天 | 社区中心 | 给玩家一个长期目标 |
| 春 13 | 复活节+草莓 | 第一个节日+全新作物 |
| 第1次进矿 | 战斗、采矿玩法 | 游戏世界深度开启 |
| 第1个鸡舍 | 动物系统 | 被动收入 |
| 升级洒水器 | 自动化 | 时间解放 |
| 夏季 | 完全不同的一组作物/鱼 | "我熟悉的一切都要重新学" |
| 社区中心 Progress | 每个Bundle解锁新区域/新奖励 | 推动探索 |
| 第二年 | Kent 回归+新内容 | 世界变化了 |
| 第三年 | 祖父评分 | 最终评价+完美目标 |
| 姜岛(1.5) | 新大陆+新系统+新玩法 | 终局内容 |
| 精通(1.6) | 技能毕业后还有追求 | 终局再延长 |

**这个解锁曲线的核心原则：**
> **每次玩家觉得"差不多了"的时候，游戏就给你一个新的系统。**  
> 种地闷了？去挖矿。挖矿腻了？钓鱼。钓鱼烦了？去追NPC。恋爱好感满了？有姜岛。姜岛逛完了？有Qi挑战。

## 18.3 缺失的"强制性失败"

星露谷没有：
- **Game Over**（你永远可以重来，死亡只掉物品，不会删档）
- **时间限制**（社区中心不限时，Joja路线不限时）
- **强制任务**（每个任务都可以忽略）
- **等级门槛**（所有内容都可以低等级完成，只是难一点）

这降低了焦虑感，让玩家"自愿地"参与游戏内容。

## 18.4 "进步可视化"系统

星露谷让每一点进步都可见：

| 进步 | 可视化方式 |
|------|-----------|
| 农场旧貌→新颜 | 画面直接对比 |
| 工具升级 | 锄头从破到金灿灿 ✨ |
| 技能升级 | 屏幕弹出 + 新配方 |
| NPC好感 | 从冷冰冰到送你礼物 |
| 社区中心 | 金闪闪的动画每完成一个 Bundle |
| 金钱 | 不断增长的数字 |

# 第十九部分：游戏引擎与实现技术要素

## 19.1 技术概览

| 层面 | 技术 |
|------|------|
| 引擎 | MonoGame（XNA 4.0 开源实现） |
| 语言 | C# |
| 渲染 | 2D SpriteBatch，DirectX/OpenGL |
| 地图 | Tile-based，每个 Tile 16×16 像素（渲染时 ×2/×3 缩放） |
| 存档 | 基于 XML 序列化，每天睡觉保存 |
| Mod加载 | SMAPI（Stardew Modding API，第三方） |
| 多人 | 基于 C# 异步 + 网络流（Galaxy/Steam/局域网） |
| 性能 | 支持超低配设备 |

## 19.2 渲染管线

1. 全游戏渲染到 RenderTarget2D（固定分辨率）
2. 然后缩放至屏幕分辨率（点采样 = 邻近插值，保持像素感）
3. 世界缩放因子母：4（每个像素 4×4 块）
4. 使用内置的 SpriteSortMode.Texture 动态批次合并 Sprite

## 19.3 核心架构模式

```
- Game1.cs — 主入口，持有所有 Subsystem 引用
- 子系统包括：
  - Location 类 → Farm / Town / Mine / ...（通过继承）
  - Farmer 类（玩家状态）
  - NPC 类（AI+日程）
  - Crop / Animal / Furniture 等数据类
  - 事件系统（Event）— 控制心事件/剧情动画
- 所有数据使用 XNB 格式（管道处理后的纹理/地图/字体）
```

## 22.3 网络架构设计

### 方案对比

| 架构 | 原理 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **Listen Server** | 游戏客户端内置服务器，主机既玩又当服务器 | 零额外部署成本、局域网延迟极低 | 主机退出游戏结束、主机有轻微额外负载 | ⭐⭐⭐⭐⭐ |

#### 路径 A：纯 WebSocket（推荐，与 TS 全栈无缝衔接）

**嵌入方式（核心设计）：**

```typescript
// 服务端作为主机客户端的一个"子系统"
// 不需要独立部署，启动游戏时选择"新建联机房间"即启动内嵌服务端

class GameServer {
  private io: Server;                          // socket.io 实例
  private gameState: GameState;                // 共享游戏状态（由主机控制）
  private clients: Map<string, PlayerSession>; // 连接的玩家
  
  constructor(gameState: GameState) {
    this.io = new Server({
      maxConnections: 4,                       // 最多4人
      pingInterval: 2000,                      // 2秒心跳
      pingTimeout: 5000,                       // 5秒超时断开
    });
    this.gameState = gameState;
    this.clients = new Map();
    
    this.io.on('connection', this.handleConnection);
  }
  
  // 核心同步方法：主机将状态广播给所有客户端
  broadcastState(): void {
    const snapshot: GameStateSnapshot = {
      players: this.getPlayerPositions(),
      time: this.gameState.time,
      weather: this.gameState.weather,
      crops: this.getChangedCrops(),           // 增量更新
      animals: this.getChangedAnimals(),        // 增量更新
      npcs: this.getNPCPositions(),
    };
    this.io.emit('state_update', snapshot);
  }
}
```

#### 路径 B：WebRTC（点对点，延迟最低但实现复杂）

```
┌─ X200Mini ────┐         ┌─ R9000P ──────┐
│  WebRTC peer  │ ◄───────► WebRTC peer   │
```

### 增量更新（Delta Compression）

**为什么重要：** 农场可能有 1000+ 个地块，但一次同步只需发送变化的几个。

```typescript
// 全量 vs 增量
interface CropTileFull {
  x: number; y: number;
  fertilized: boolean;
  watered: boolean;
  cropId: string;
  growthStage: number;
  quality: number;
  // ...每个地块约 50 字节，1000 个地块 = 50KB/次
}

interface CropTileDelta {
  x: number; y: number;              // 2 bytes each → 4 bytes
  changed: number;                   // bitmask: 哪些字段变了
  // 仅在 changed 对应位为 1 时发送：
  watered?: boolean;                 // 1 bit
  growthStage?: number;              // 0-5 → 3 bits
  // ...每次同步通常只有 5-20 个地块变化
}
// 增量更新 ≈ 200-500 字节/次 vs 全量 50KB/次
```

### 5.2 时间同步（最关键的系统）

星露谷的联机时间机制是**主机控制时间**。你必须实现同样的模式：

```typescript
// 时间控制器（仅在主机上运行）
class TimeController {
  private hour: number = 6;        // 6:00 AM
  private minute: number = 0;
  private speedMultiplier: number = 1;  // 1x 正常, 10x 睡觉加速
  private isPaused: boolean = false;
  
  // 时间推进（每秒调用）
  tick(deltaSeconds: number): void {
    if (this.isPaused) return;
    
    const gameMinutesPerTick = deltaSeconds * 60 * this.speedMultiplier;
    this.minute += gameMinutesPerTick;
    
    // 进位逻辑
    while (this.minute >= 60) {
      this.minute -= 60;
      this.hour += 1;
    }
    
    // 凌晨 2:00→6:00 睡觉结算
    if (this.hour >= 26) {  // 游戏内凌晨2:00 = 26:00
      this.triggerSleepEvent();
    }
    
    // 同步到所有客户端
    this.broadcastTime();
  }
  
  // 所有玩家都睡觉时加速
  allPlayersSleeping(): void {
    this.speedMultiplier = 10;  // 10倍速
  }
  
  // 任一玩家醒来时恢复
  anyPlayerWakeUp(): void {
    this.speedMultiplier = 1;
  }
}
```

### 5.3 同步冲突处理

```typescript
// 示例：同一块地两个人同时操作
// 主机仲裁规则：
// 1. 先收到谁的操作，谁生效（主机本地时间优先）
// 2. 第二种操作被拒绝，客户端收到拒绝消息后回滚动画
// 3. 客户端显示 "该地块已被操作" 的短暂提示

class ActionResolver {
  resolveTileAction(tileId: string, action: ActionData, playerId: string): ActionResult {
    const lastAction = this.actionLog.get(tileId);
    
    if (lastAction && (Date.now() - lastAction.timestamp < 200)) {
      // 200ms 内有两个操作 → 冲突
      return {
        success: false,
        message: '该地块正在被操作',
        fallbackState: this.currentState.getTile(tileId),
      };
    }
    
    // 记录操作
    this.actionLog.set(tileId, { playerId, action, timestamp: Date.now() });
    return { success: true };
  }
}
```

```typescript
// 断线重连策略
class ReconnectionManager {
  private disconnectedPlayers: Map<string, DisconnectedPlayer> = new Map();
  
  onPlayerDisconnect(playerId: string): void {
    // 玩家掉线，保留其状态5分钟
    const player = this.players.get(playerId);
    this.disconnectedPlayers.set(playerId, {
      player,
      disconnectTime: Date.now(),
      inventory: player.inventory,           // 保留背包
      position: player.position,             // 记录最后位置
    });
    
    // 角色原地保留（不可交互）
    this.spawnGhost(player.position);
    
    // 5分钟后清理
    setTimeout(() => {
      this.cleanupDisconnectedPlayer(playerId);
    }, 300000);
    
    // 通知其他玩家 "XXX 掉线了"
    this.broadcast({ type: 'player_disconnected', playerId });
  }
  
  onPlayerReconnect(playerId: string, newSession: Session): boolean {
    const ghost = this.disconnectedPlayers.get(playerId);
    if (!ghost) return false;  // 超时无法重连
    
    // 恢复玩家状态
    const restoredPlayer = {
      ...ghost.player,
      inventory: ghost.inventory,
      position: ghost.position,
    };
    
    this.players.set(playerId, restoredPlayer);
    this.disconnectedPlayers.delete(playerId);
    
    // 恢复成功
    this.broadcast({ type: 'player_reconnected', playerId });
    return true;
  }
}
```

## 23.4 像素游戏性能优化技术清单

### 渲染优化

| 技术 | 原理 | 优化幅度 | 实现难度 |
|------|------|----------|----------|
| **纹理图集（Texture Atlas）** | 将所有小图合并为一张大图，减少 GPU 绘制调用 | 10-50 倍 | 低 |
| **精灵批次（Sprite Batching）** | 同纹理的 Sprite 合为一次 Draw Call | 5-20 倍 | 框架自带 |
| **视锥裁剪（Frustum Culling）** | 只渲染屏幕可见范围内的物体 | 2-10 倍 | 中 |
| **TileMap 动态加载** | 只加载当前屏幕区域的地块 | 5-20 倍 | 中 |
| **LOD 系统（LOD）** | 远处物简化渲染 | 1.5-3 倍 | 中 |
| **离屏渲染缓存** | 复杂静态界面渲染到缓存，避免逐帧重绘 | 2-5 倍 | 中 |
| **Power of Two 纹理** | 纹理宽高为 2 的幂，GPU 处理更快 | 1.2-2 倍 | 低 |

### 纹理图集实现（像素游戏必做）

### 对象池（Object Pooling）

```typescript
// 避免频繁创建/销毁导致的 GC 卡顿
// 适用于：粒子、掉落物、NPC、临时 UI 元素

class ObjectPool<T extends Poolable> {
  private pool: T[] = [];
  private factory: () => T;
  
  constructor(factory: () => T, initialSize: number) {
    this.factory = factory;
    // 预创建对象
    for (let i = 0; i < initialSize; i++) {
      const obj = factory();
      obj.deactivate();
      this.pool.push(obj);
    }
  }
  
  acquire(): T {
    const obj = this.pool.find(o => !o.isActive);
    if (obj) {
      obj.activate();
      return obj;
    }
    // 池不够时再创建（兜底，但不应该频繁触发）
    const newObj = this.factory();
    newObj.activate();
    this.pool.push(newObj);
    return newObj;
  }
  
  release(obj: T): void {
    obj.deactivate();
  }
}

// 使用示例：矿洞掉落物
const dropPool = new ObjectPool<ItemDrop>(() => new ItemDrop(), 50);
```

```typescript
// Phaser 3 原生支持触屏事件，但需要针对像素游戏优化：

class MobileInputManager {
  private scene: Phaser.Scene;
  private joystick: VirtualJoystick;
  
  constructor(scene: Phaser.Scene) {
    this.scene = scene;
    
    // 检测是否触屏设备
    if (!scene.sys.game.device.input.touch) {
      return; // PC 端不启用触屏逻辑
    }
    
    // 增加所有可交互目标的碰撞体（让手指容易点到）
    this.expandInteractiveAreas();
    
    // 注册触屏手势
    this.setupTouchGestures();
  }
  
  private expandInteractiveAreas(): void {
    // 所有可点击物体的点击区域放大到 1.5 倍
    // （不影响视觉大小，只影响点击检测区域）
    scene.children.each((child: Phaser.GameObjects.GameObject) => {
      if (child.input && child.input.hitArea) {
        const hitArea = child.input.hitArea as Phaser.Geom.Rectangle;
        hitArea.width *= 1.5;
        hitArea.height *= 1.5;
      }
    });
  }
  
  private setupTouchGestures(): void {
    // 双指缩放（地图缩放）
    this.scene.input.on('wheel', (_pointer, _gameObjects, deltaX, deltaY) => {
      // 触屏双指手势映射到滚轮事件
      // Phaser 3 自动将触屏双指操作转为 zoom 事件
    });
    
    // 长按连续操作（连续耕地）
    this.scene.input.on('pointerdown', (pointer: Phaser.Input.Pointer) => {
      if (pointer.wasTouch) {
        this.startLongPressTimer(pointer);
      }
    });
    
    this.scene.input.on('pointerup', () => {
      this.cancelLongPressTimer();
    });
  }
  
  // 智能目标选择：触屏点击时自动选择最近的交互目标
  getNearestInteractable(touchX: number, touchY: number): GameObject | null {
    const threshold = 64; // 64px 范围内找目标
    let nearest: GameObject | null = null;
    let minDist = threshold;
    
    this.interactables.forEach(obj => {
      const dist = Phaser.Math.Distance.Between(touchX, touchY, obj.x, obj.y);
      if (dist < minDist) {
        minDist = dist;
        nearest = obj;
      }
    });
    
    return nearest;
  }
}
```

1. **点击区域 >= 44×44px**（苹果 HIG 标准手指触控大小）

3. **工具自动切换**（走到耕地前自动换成锄头，走到未浇水作物前自动换成水壶）

5. **横竖屏都适配**（强制横屏锁定，游戏 UI 不旋转）
