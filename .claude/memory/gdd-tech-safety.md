# GDD 技术与安全规范（v3.3 更新版摘录）

## 代码规范（不可变更）
- 全栈 TypeScript，禁止 any（用 unknown 替代）
- 所有变量/参数/返回值显式类型声明
- 命名：camelCase（变量/函数），PascalCase（类/接口/类型）
- 文件命名：kebab-case.ts
- 单文件 ≤ 300 行（纯数据/配置文件可放宽至 500 行，需顶部注释说明）
- 单函数 ≤ 50 行
- 注释：中文 JSDoc

## 架构规范（不可变更）
1. **系统类模式**：每个核心系统实现为独立 System 类（FarmingSystem、TimeSystem 等），Scene 仅负责生命周期编排
2. **状态管理**：全局 GameState store，所有系统通过它读写共享数据（存档序列化、联机同步、调试快照）
3. **错误隔离**：每个 System 有独立 try/catch 边界，单系统崩溃降级为「该功能暂不可用」
4. **系统间通信**：通过 EventBus 事件总线，禁止系统间直接方法调用

## 工程工具链
- ESLint + @typescript-eslint/strict（强制禁止 any、强制返回类型）
- Prettier（统一代码风格）
- Vitest（单元测试）

## 安全防护（10 维熔断保护）

> 参数分类：HARD_LIMIT（引擎/技术绝对限制）vs BASELINE（首发基准值，可随版本调整，定义在 GameConfig）

### 11.4.1 游戏循环安全
- 帧时间上限：100ms/帧（HARD_LIMIT），超限跳过剩余
- 迭代上限：10,000 次/update（HARD_LIMIT）
- 看门狗：5秒检查，3秒阻塞 → 第1次暂停+提示玩家 → 第2次超时才重启场景（保存状态到临时存档）

### 11.4.2 渲染安全（BASELINE）
- 精灵上限：全局 5,000 / 每层 1,000
- 粒子上限：每发射器 200 / 同时 20 个发射器
- 纹理内存上限：PC 256MB / 手机 160MB
- Tile 裁剪：视口外 +2 tile 缓冲，单帧 ≤ 10,000 tile（HARD_LIMIT）

### 11.4.3 网络安全
- 速率限制：60 消息/秒，50KB/秒上行（HARD_LIMIT）
- 单消息上限：16KB（HARD_LIMIT）
- 连接超时：初次 10s，空闲 5min 断开（HARD_LIMIT）
- 状态验证：位置合法性、最大移速 8 tile/s、金钱变动动态上限 max(50000, playerMoney*0.5)

### 11.4.4 内存安全
- 场景切换：清理纹理缓存/音效/Tween + 平台自适应 GC（浏览器:释放引用, Tauri:global.gc()）
- 资源加载超时：15s/资源（HARD_LIMIT）
- 缓存上限（BASELINE）：PC 纹理 128MB/音频 64MB/数据 500 条 | 手机 纹理 80MB/音频 32MB/数据 300 条
- 对象池：最多 50 池 / 每池 500 对象（HARD_LIMIT）

### 11.4.5 数据安全
- 原子写入 + sha256 校验 + 覆盖前备份（HARD_LIMIT）
- 数值边界（BASELINE）：金钱 0-999,999,999 / 体力 0-999 / HP 0-999 / 好感 0-2,500 / 堆叠 0-999 / 技能 0-10
- 分级自动修复：值越界→autoClamp / 字段缺失→fillDefault / 类型不匹配→提示用户 / 结构损坏→提示用户
- 存档限流：最小间隔 30s、单日上限 100 次
- 恢复：自动存档恢复 / 3 个备份槽 / 损坏时提示用户

### 11.4.6 状态机安全
- 非法状态转换：回退并记录
- NPC 日程异常：使用每个地图的 safeSpawnPoints.npcDefault（不再硬编码 0,0）
- 任务一致性：读档检查 + 自动修复
- 物品 ID：白名单验证，拒绝未知 ID

### 11.4.7 文件 I/O 安全
- 单文件 ≤ 10MB（HARD_LIMIT）
- 允许扩展名：.json .png .wav .ogg .mp3 .tsx .tmx
- 设置文件：验证→损坏用默认→自动恢复

### 11.4.8 联机安全
- 玩家硬上限 8（设计 4）（HARD_LIMIT）
- 分级降载：降频 → 踢带宽最高者 → 踢最新连接
- 消息队列：最多 1,000 条 / 每帧处理 10ms（HARD_LIMIT）
- 作弊防护：操作 ≤ 20 次/秒 / 50 次标记可疑（BASELINE）

### 11.4.9 输入安全（BASELINE）
- 按键防抖 150ms / 商店购买 ≤ 10 次/秒 / 对话选择 ≤ 5 次/秒 / 工具使用 ≤ 5 次/秒

### 11.4.10 WebGL 上下文安全
- context lost → 暂停渲染 → 等待 restored → 重新上传纹理 → 恢复场景
- 最多 3 次恢复尝试，失败则显示错误返回主菜单
- 恢复前保存游戏状态到临时存档

## 页面可见性处理
- 页面隐藏：暂停游戏循环 / 停止音频 / 降低网络同步频率
- 页面恢复：重新校准时间 / 恢复音频 / 恢复网络 / 若 context lost 则重载场景

## 内存泄漏检测
- 每 5 分钟记录内存水位
- 连续 3 次上升超 50MB 触发告警

## 平台差异化资源缓存（BASELINE）
| 类别 | PC | 手机 |
|------|:--:|:----:|
| 纹理缓存 | ≤ 128MB | ≤ 80MB |
| 音频缓存 | ≤ 64MB | ≤ 32MB |
| 数据条目 | ≤ 500 | ≤ 300 |

## 错误恢复策略（8类）
| 异常 | 检测 | 恢复 | 回退 |
|------|------|------|------|
| 存档损坏 | checksum/json/字段/类型/值域 | 分级：值越界autoClamp/字段缺失fillDefault/类型不匹配提示 | 用自动存档 |
| 网络断开 | 超时/心跳/拒绝 | 自动重连/5秒重试 | 离线模式 |
| 资源加载失败 | 超时/404/解码错误 | 跳过/重试/占位纹理 | 缺失纹理占位 |
| 渲染崩溃 | WebGL丢失/内存溢出 | WebGL：暂停→等restored→重上传→恢复 / 内存：降质→清理→重载 | 错误提示退出 |
| 任务状态异常 | 目标数/前置/标记 | 自动修复/重置/标记 | 记录并跳过 |
| 玩家位置异常 | 越界/碰撞/地下 | 传送出生点/上一安全点 | 地图 emergencyExit |
| 时间异常 | 倒退/跳跃/跳日 | 回退有效值/截断 | 强制睡眠存档 |
| 系统崩溃 | 未捕获异常/超时 | 降级该系统 | 保存并提示重启 |

## 开发风险清单
| 风险 | 可能性 | 影响 | 缓解 |
|------|:------:|:----:|------|
| 联机延迟过高 | 中 | 高 | 客户端预测+插值 |
| 存档损坏丢档 | 低 | 极高 | 原子写入+校验+备份+分级修复 |
| 大量作物降帧 | 中 | 中 | 渲染裁剪+精灵上限 |
| 网络消息洪水 | 低 | 高 | 速率限制+队列保护 |
| 数据溢出 | 低 | 中 | 数值边界保护+动态上限 |
| NPC日程异常 | 中 | 中 | safeSpawnPoints回退 |
| 手机内存不足 | 中 | 高 | 平台差异化缓存+场景清理 |
| 多人抢地冲突 | 低 | 中 | 主机仲裁+客户端回滚 |
| 长时间内存泄漏 | 中 | 高 | 内存水位监控+定时检测 |
| 手机WebGL丢失 | 中 | 高 | context restored自动恢复 |
| 系统崩溃拖垮全局 | 低 | 极高 | 错误隔离+降级策略 |

## 游戏配置集中管理（→ 11.8）
- 所有 BASELINE 参数统一定义在 `src/config/game-config.ts`
- 按模块分区：render / cache / economy / player / network / input / memory / save
- 业务代码只通过 GameConfig 引用，禁止硬编码
- 平台自适应加载：detectPlatform() 后合并 PC/Mobile overrides
